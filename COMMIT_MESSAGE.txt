Implement comprehensive metadata preservation and transformation framework with
support for timestamps, permissions, ownership, extended attributes, and path
transformations. Enables cross-platform migrations, reproducible builds, and
privacy-aware backups.

## Features

### Core Metadata Support
- Complete file metadata structure supporting:
  - Timestamps (atime, mtime, ctime)
  - Unix permissions (mode bits)
  - Ownership (UID/GID on Unix)
  - Extended attributes (xattrs) via feature flag
  - Windows file attributes
  - ACL entries (structure defined, ready for implementation)
- Manifest integration for metadata serialization
- Platform-specific handling (Unix/Windows/macOS)

### Backend Abstraction Extensions
- Added 5 new async trait methods to Backend:
  - `set_permissions()` - Set Unix permission bits
  - `set_timestamps()` - Set access and modification times
  - `get_xattrs()` / `set_xattrs()` - Extended attribute operations
  - `set_ownership()` - Set owner and group
- Default implementations return Unsupported for graceful degradation
- Full LocalBackend implementation with:
  - Async/blocking bridging via tokio::spawn_blocking
  - Platform-specific conditional compilation
  - Comprehensive error handling

### Transformation Framework
- Regex-based path transformations (sed-like syntax)
- Case conversion (lowercase, uppercase, titlecase)
- Metadata filtering:
  - Strip xattrs (all or by pattern)
  - Strip ownership/permissions
  - Normalize timestamps (for reproducible builds)
- CLI string parser for easy configuration
- Builder pattern for programmatic use

### Orchestration Layer
- MetadataPreserver for high-level coordination
- Support for both local and backend-aware operations
- Configurable strictness (warn vs. fail on errors)
- Metadata verification capabilities
- Statistics tracking (MetadataStats)

### CLI Integration
- New command-line options:
  - `--preserve-metadata` / `-p` - Basic preservation
  - `--preserve=FLAGS` - Selective preservation
  - `--transform=CONFIG` - Apply transformations
  - `--strict-metadata` - Fail on any error
  - `--verify-metadata` - Post-transfer verification
- Configuration file support (orbit.toml)

## Examples

```bash
# Basic preservation
orbit --source /data --dest /backup -p --recursive

# Selective preservation
orbit --source /data --dest /backup --preserve=times,perms,owners

# With transformations
orbit --source /photos --dest /archive \
  --preserve=all \
  --transform="rename:s/IMG_/photo_/,case:lower"

# Strip sensitive metadata for cloud
orbit --source /data --dest s3://bucket/data \
  --preserve=times,perms \
  --transform="strip:ownership,strip:xattrs"
```
